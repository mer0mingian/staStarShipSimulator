# Open Questions from M8.2 Rules Analysis

This document captures unresolved ambiguities and design decisions arising from the architectural review of core rulebook chapters (Ch 4, 5, 6, 7, 8, 9, 11) against the VTT implementation constraints.

## General Rule Interpretation & VTT Mapping Questions

| Source Chapter | Question Summary | Required Clarification / Decision Point |
| :---: | :--- | :--- |
| **Ch 4, 8** | **Stress/Fitness Mapping:** What attribute/value defines a Starship's maximum Stress Capacity (equivalent to a Character's Fitness)? | Is it a fixed value, derived from Engineering/Control, or a new property on the `VTTShipRecord`? |
| **Ch 4, 8, 9** | **Initiative Keeping:** Does the ability to spend Momentum (2) to "Keep the Initiative" apply to Starship Combat rounds, and if so, is the cost based on Crew Size or a flat amount? | Confirms the rule's scope (Personal/Social/Starship) is essential for implementation. |
| **Ch 4, 11** | **NPC Stress/Determination:** Ch 11 states NPCs use Threat instead of Determination, while `objects.md` lists a generic Stress Track for all Characters. | Is there explicit logic to *suppress* any Species Traits that modify Stress/Determination for NPC objects? |
| **Ch 9** | **Reinforcement Cost:** How is the cost determined to introduce a new enemy ship via Threat Spend? | Is the cost based on the enemy ship's **Scale** or **Crew Quality**? |
| **Ch 7, Action Framework** | **Task vs. Action:** Should the term "Task" from Chapter 7 be retired in the VTT implementation in favor of configured **Actions** or **Extended Tasks**? | Defines how generic non-action challenges will be resolved. |
| **Ch 11** | **NPC Trait Suppression:** Are species traits that modify Stress/Determination automatically ignored for NPCs, or must the VTT explicitly check and remove them? | Dictates validation logic for NPC object creation. |

## NPC Implementation Questions (Ch 11)

| Source Chapter | Question Summary | Required Clarification / Decision Point |
| :---: | :--- | :--- |
| **Ch 11.1** | **NPC Attribute Distribution:** Must the VTT enforce the exact numerical spread (e.g., 9, 9, 8, 8, 7, 7) when generating Major/Notable NPCs, or only check the final score sums? | Impacts NPC generation tool complexity. |
| **Ch 11.1** | **Special Rule Limitations:** For rules like "PROFICIENCY," how should the system structure the required "particular task" limitation—as a simple string or a predicate language? | Impacts future design for extending NPC capabilities. |
| **Ch 11.3** | **Crew Quality Application:** For NPC ships, should the VTT use the fixed Crew Quality rating (e.g., Attribute 10 for Talented) as the baseline for all rolls, or simulate the underlying department structure? | Defines the complexity of NPC Ship action resolution.
