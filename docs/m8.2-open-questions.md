# Open Questions from M8.2 Rules Analysis

This document captures unresolved ambiguities and design decisions arising from the architectural review of core rulebook chapters (Ch 4, 5, 6, 7, 8, 9, 11) against the VTT implementation constraints.

## General Rule Interpretation & VTT Mapping Questions

| Source Chapter | Question Summary | Required Clarification / Decision Point | Response |
| :---: | :--- | :--- | :--- |
| **Ch 4, 8** | **Stress/Fitness Mapping:** What attribute/value defines a Starship's maximum Stress Capacity (equivalent to a Character's Fitness)? | Is it a fixed value, derived from Engineering/Control, or a new property on the `VTTShipRecord`? | A ship has the shields track - which is derived from it's Scale, Spaceframe (i.e. class) and Mission Profile. You also need to consider Resistance. Beyond that the ship has a max number of Breaches (Ship Trait with Potency, representing system damage) to a system it can handle before it is being destryoed and the "Shaken" (Ship Trait). Ships can handle a number of breaches in total equal to their scale OR a number of breaches to a single system equal to half the ships scale - whichever is smaller.  Read this in Chapter 5 and 8.3. |
| **Ch 4, 8, 9** | **Initiative Keeping:** Does the ability to spend Momentum (2) to "Keep the Initiative" apply to Starship Combat rounds, and if so, is the cost based on Crew Size or a flat amount? | Confirms the rule's scope (Personal/Social/Starship) is essential for implementation. | Look this up in Chapter 8.3 under "Prepare for Combat". |
| **Ch 4, 11** | **NPC Stress/Determination:** Ch 11 states NPCs use Threat instead of Determination, while `objects.md` lists a generic Stress Track for all Characters. | Is there explicit logic to *suppress* any Species Traits that modify Stress/Determination for NPC objects? | No, this depends on the type of NPC or Supporting Character. Supporting Characters receive a Stress Track and Determination with their first Value. Major and Notable NPCs have a Stress Track, but Minor ones do not. Also note, that NPCs can have Personal Threat pools (see Chapter 11.1 "The Cast of Adversaries"). |
| **Ch 9** | **Reinforcement Cost:** How is the cost determined to introduce a new enemy ship via Threat Spend? | Is the cost based on the enemy ship's **Scale** or **Crew Quality**? | This is 1 Threat per Scale unit of the new ship. See Chapter 9, "Spending Threat" |
| **Ch 7, Action Framework** | **Task vs. Action:** Should the term "Task" from Chapter 7 be retired in the VTT implementation in favor of configured **Actions** or **Extended Tasks**? | Defines how generic non-action challenges will be resolved. | No, Tasks are all dice-rolls, but specifically the ones that Challenges (chained tasks) consist of. Actions are related to your Crew Station on the Bridge during Starship Combat only. | 
| **Ch 11** | **NPC Trait Suppression:** Are species traits that modify Stress/Determination automatically ignored for NPCs, or must the VTT explicitly check and remove them? | Dictates validation logic for NPC object creation. | These should be ignored neither for NPCs nor PCs. But we can push them for future ideas and simply allow an overwrite by the GM/creating Player for now. If we are greedy we can notify of a Species Ability of Special Rule, Role, or Talent that they refer to a Stress Track. |

Take-away: Please create an object for Talent (and Special Rules for NPCs/Creatures, and Role Benefits, and Species Abilities) that allows to execute associated rules for future implementation. E.g. if a Species Ability changes the Stress Track, create a plan how to represent that in the code for character creation. The same categories may be relevant for implementing game logic for Traits.

## NPC Implementation Questions (Ch 11)

| Source Chapter | Question Summary | Required Clarification / Decision Point | Response |
| :---: | :--- | :--- | :--- |
| **Ch 11.1** | **NPC Attribute Distribution:** Must the VTT enforce the exact numerical spread (e.g., 9, 9, 8, 8, 7, 7) when generating Major/Notable NPCs, or only check the final score sums? | Impacts NPC generation tool complexity. | The numerical spread must be checked or enforced by design of the NPC creation dialogue. In the latter case, you only need to check the sum. |
| **Ch 11.1** | **Special Rule Limitations:** For rules like "PROFICIENCY," how should the system structure the required "particular task" limitation—as a simple string or a predicate language? | Impacts future design for extending NPC capabilities. | This relates to the Take-away from above. Plan the concepts for a predicate language, but rely on strings for now. Note it as future development in the docs. |
| **Ch 11.3** | **Crew Quality Application:** For NPC ships, should the VTT use the fixed Crew Quality rating (e.g., Attribute 10 for Talented) as the baseline for all rolls, or simulate the underlying department structure? | Defines the complexity of NPC Ship action resolution. | Make this dependent on whether the player has assigned an NPC to the Station that is executing the task. If yes, use the NPC. If no, use the Crew Qaulity.  |
